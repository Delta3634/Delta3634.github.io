T2 两数相加

事实证明，我的C++水平还是很差，感觉和初学者差不多。。。😅😅😅😅😅

想到思路之后，自己试着写了一点，然后把代码和思路丢给豆包让它改。😅 链表的用法完全忘记了，在那里搞得那么复杂，很不好啊啊啊啊。

我的思路——先遍历，再相加，最终构建链表

  • 定义两个vector用来获取当前链表中的数字；

  • 按位相加（刚开始没有考虑到对齐，进位的处理我也没想到）

    • 对齐
    • 相加（考虑进位）
    • 存储当前结果、更新进位

  • 将结果转换为逆序链表（当前结果为逆序，所以直接for就好）

    • 这里复习到了 虚拟头结点

官方题解——直接遍历链表并同步构建结果链表

  • 定义两个ListNode节点

    • head： 指向结果链表的第一个节点（最终返回此指针）
    • tail： 始终指向结果链表的最后一个节点，用于高效添加新节点

  • 遍历链表（对齐并按位相加）

  • 构建结果链表

    • 第一次循环时，head 和 tail均为 nullptr，所以同时进行初始化
    • 后续循环：通过tail->next 添加新节点，保持tail始终指向最后一个节点

  • 更新进位、移动指针

  • 处理最后的剩余进位

学到的：虚拟头结点 ListNode* head = new ListNode(0)

  • 返回的时候就return head->next
  • 在写完整代码的时候，return完要释放内存：delete head

T5 最长回文子串

双指针

双指针也是一种解题思路，并非都是真正的ptr。

面向：

  • 数组、字符串（整数索引）、链表（真正的ptr）：减少遍历次数；
  • 滑动窗口：左右指针动态维护窗口；
  • 回文相关：从中心向两端（确定回文长度）/ 从两段向中间（验证回文）；
  • 链表的特殊操作：高效遍历 & 特殊定位；
  • 归并与排序相关

函数 功能描述 示例
size() / length() 返回字符串长度（字符个数） s.size() → 5（若 s = "hello"）
empty() 判断字符串是否为空（长度为 0） s.empty() → true（若 s = ""）
substr(pos, len) 截取子串：从索引 pos 开始，长度为 len（若 len 省略则取到末尾） s.substr(1,3) → “ell”（s="hello"）
c_str() 返回字符串的 C 风格指针（const char*），用于兼容 C 语言函数 printf("%s", s.c_str())
at(pos) 返回索引 pos 处的字符（带越界检查，比 s[pos] 更安全） s.at(1) → ‘e’（s="hello"）
front() / back() 返回第一个 / 最后一个字符 s.front() → ‘h’（s="hello"）
push_back(c) 在字符串末尾添加字符 c s.push_back('!') → “hello!”
pop_back() 删除字符串末尾的字符 s.pop_back()（s="hello!" → “hello”） s.pop_back（）（s=“你好！”→“你好”）
append(str) 在末尾拼接字符串 str（类似 + 运算符） s.append("world") → “helloworld”
insert(pos, str) 在索引 pos 处插入字符串 str s.insert(2, "xx")（s="ab" → “axxb”）
erase(pos, len) 从索引 pos 开始删除 len 个字符（len 省略则删除到末尾） s.erase(1,2)（s="hello" → “hlo”）
find(str) 查找 str 在字符串中第一次出现的索引，未找到返回 string::npos s.find("ll") → 2（s="hello"）
rfind(str) 查找 str 在字符串中最后一次出现的索引，未找到返回 string::npos s.rfind("l") → 3（s="hello"）
replace(pos, len, str) 从索引 pos 开始，用 str 替换 len 个字符 s.replace(1,2,"xx")（s="hello" → “hxxlo”）
compare(str) 比较当前字符串与 str，返回 0（相等）、正数（当前更大）、负数（当前更小）

T8 字符串转换整数(atoi)

这道题也是很有意思啊，我能把我的思路写出来，问题：

  • 对字符判断的理解有问题

    • 只考虑了超出9的，还写的是整型。。

    • s[i] >= 10 是典型错误，字符的本质是 ASCII 码（比如 '0' 的 ASCII 是 48，'9' 是 57），判断是否为数字应该用 s[i] >= '0' && s[i] <= '9'，而非与数字 10 比较。

  • 对库函数的参数类型不熟悉

    程序设计留下的大坑啊🥲🥲🥲😶‍🌫️😶‍🌫️😶‍🌫️

    • 我的代码里直接写了这一句：int ans = atoi(mid);

    • 因为 atoi 需要 const char* 类型（C 风格字符串），而不是 vector<char>。如果想用容器，需要先将其转换为 C 风格字符串（比如用 &mid[0]，但需确保以 '\0' 结尾）

  • 缺少边界情况处理

    • 这一点我是真的真的完全没想到

    • 特殊情况也没考虑全

      • 没有考虑 “空格后直接出现非数字 / 符号” 的情况（比如 " abc" 应返回 0）。
      • 完全没有处理溢出（转换结果超过 INT_MAX 或 INT_MIN 时的处理）。
      • 符号只能出现一次，且必须在数字前（比如 "+-12" 应返回 0），这些特殊情况都没考虑。

学到的：isdigit

  • 功能：判断一个字符是否为十进制数字字符（即 '0' 到 '9'）。

  • 头文件：需要包含 <cctype>

  • 参数：接收一个 char 类型的字符（实际会被隐式转换为 int，通常传入字符即可）。

  • 返回值：若为数字字符，返回非零值（通常是 true）；否则返回 0（通常是 false）。

  • 示例

    • isdigit('5') → 返回 true（非零）
    • isdigit('a') → 返回 false（0）
    • isdigit('+') → 返回 false（0）
