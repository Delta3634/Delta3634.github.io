——》不具有通用性

切片

  • 动态数组
  • 静态函数作为函数参数，传参的过程是：引用传递，而且不同元素长度的动态数组他们的形参是一致的。

声明方式

  1. slice := []int{1,2,3}

  2. var slice[]int 此时slice未分配空间

  3. var slice[]int = make([]int,3) —— 用make给slice分配空间，3个空间，初始值为0

    slice := make([]int,3) 等价于上面的写法，只是这里通过:=推导出slice是一个切片

切片的追加与截取

  • 长度len：左右指针之间的距离
  • 容量cap：左指针至底层数组末尾的距离。

切片容量的追加：

  1. numbers = make([]int,3,5)：当前的切片空间为3——len，容量为5——cap

  2. 追加：numbers = apppend(numbers,1)

    追加一个元素1，如果numbers = append(numbers,2)那就是追加一个元素2了

    现在numbers的长度为4，容量仍然为5

  3. 扩容机制：如果追加的长度超过一个cap的长度，那就会默认再扩展一个cap出来（即容量增加2倍）

切片截取：

  1. s := slice[0:n]：截取的元素从第一个开始到n-1为止

十二、反射

变量的结构

type 和 value ——》称作 pair

  • type分为static type 和concrete type

reflect包

  • reflect 是 Go 提供的运行时反射机制包。
  • 可以用于在运行时检查类型和变量的值，或对结构体字段、方法进行操作。
  • 常用于 ORM、序列化、动态调用等高级编程场景。

反射三要素

  • Type：类型元信息（由 reflect.TypeOf() 获取）
  • Value：值本身（由 reflect.ValueOf() 获取）
  • Kind：底层基本类型（如 int、slice、struct）

reflect.TypeOf(x)

  • 用于获取变量的类型信息，返回 reflect.Type
  • 常用方法：
    • Name()：类型名
    • Kind()：类型的种类（int、string、struct、ptr 等）

reflect.ValueOf(x)

  • 获取变量的值，返回 reflect.Value
  • 常用方法：
    • Interface()：转成 interface{}
    • Int() / String()：获取基本类型值（前提是类型匹配）
    • Elem()：获取指针指向的值
    • SetXxx()：修改值（必须使用可寻址变量，如指针）

Kind 类型分类

  • 基本类型：reflect.Int, reflect.String, reflect.Float64 等
  • 复合类型：reflect.Struct, reflect.Slice, reflect.Map, reflect.Ptr
  • 可用 Kind() 判断类型类别

结构体标签

  • 使用 t.Field(i).Tag.Get("tag名") 获取标签值

十三、goroutine基本模型和调度设计策略

  1. 进程/线程的数量越多，切换成本也就越大，也就越浪费

    例如：100%内存——》60% 执行程序 && 40% 切换中

  2. 多线程随着同步竞争（如锁、竞争资源冲突等），开发设计变得越来越复杂。

内核进程（线程thread） 用户进程（协程 co-routine）

多个线程通过协程调度器管理多个协程

Golang对协程的处理

  1. 协程co-routinue——》Goroutinue（内存：几kB ——》 所以可以大量存在）灵活调度（可常切换）

优先放在P的本地队列中，都满了这才放在全局队列中。

一个processor只能运行一个goroutine

调度器的设计策略

  • 复用线程
    • **work stealing机制 **—— 一个P的本地队列为空，从别的P的本地队列中偷取G来运行
    • **hand off机制 **—— 某G阻塞，该对应线程睡眠/销毁，创建/唤醒一个thread继续运行该进程剩余的G
  • 利用并行
    • GOMAXPROCS限定P的个数 = CPU核数 /2
  • 抢占
    • G和CPU绑定的时间最大为10ms，超过此时间，别的G会抢占CPU
  • 全局G对列
    • work stealing机制，从全局偷取

退出当前Goroutine：runtime.Goexit()

  • 关闭channel后，可以继续从channel接收数据
